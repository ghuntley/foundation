End-to-end to testing

Namespace makes it trivial to write end-to-end tests. The same server definitions you are already use for development and production are also used for testing purposes, leaving you only have to write the test specification itself.

In a test, Namespace instantiates the Servers under Test (and their dependencies) in an ephemeral environment, and runs a test driver you provide, to validate that stack. The typical test driver issues API calls against the stack, to validate a particular scenario.

To get started, create a new directory, e.g. service/test, and create a new file test.cue with your test specification:

import "namespacelabs.dev/foundation/std/fn"

test: fn.#Test & {
    // The name of the test, for debugging purposes.
    name: "test-write-workflow"

    binary: {
        // How the test driver is built. This examples uses Go, but test
        // drivers can be written in any language.
        from: go_package: "."
    }

    fixture: {
        // The package name of the server being tested.
        sut: "github.com/username/module/api/server"
    }
}

And then write your test, in service/test/test.go:

package main

import (
    "context"

    "namespacelabs.dev/foundation/testing"
    "github.com/username/module/api/service"
)

func main() {
    testing.Do(func(ctx context.Context, t testing.Test) error {
        endpoint := t.MustEndpoint("github.com/username/module/api/service", "service")

        conn, err := t.Connect(ctx, endpoint)
        if err != nil {
            return err
        }

        cli := service.NewEchoServiceClient(conn)

        // Issue an API call against the service.
        if _, err = cli.Echo(ctx, &service.EchoRequest{
            // ...
        }); err != nil {
            return err
        }

        // Test passed!
        return nil
    })
}
