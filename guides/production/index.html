Deploying to Production

Running your services and servers in a real production environment is no different than what you've experienced so far with development and testing.

Although Namespace is provider agnostic, we've optimized the onboarding experience to AWS users.

Deploying to AWS EKSâ€‹

Before we can deploy your application, you need to let Namespace know which EKS cluster to deploy to. This presupposes that you've previously created such a cluster (see section below for an example).

ns prepare eks --env=prod --aws_profile={aws_profile_name} {cluster_name}

Where

  • {aws_profile_name} refers to the locally configured aws CLI profile we should be using.
  • {cluster_name} is the name of EKS cluster we should deploy to.

After preparing your EKS cluster, you can run:

ns deploy --env=prod
Not production ready

We don't yet recommend Namespace for real production use.

Setting up AWS SSOâ€‹

An AWS SSO account represents a person with access to a set of AWS accounts, where each AWS account is an aggregation of AWS resources. After you login with SSO, you have the opportunity to act as any of the AWS accounts you have access to, either through the management console, or using programmatic credentials.

It's prudent to note that programmatic access is session based, and the credentials you get in the SSO portal are valid for 12 hours (then they need to be refreshed).

Create the EKS Clusterâ€‹

Install eksctl if needed.

Please ensure that you specify --profile [AWS-PROFILE-NAME] if you have set up an AWS account profile.

# Create cluster "zerotwo" in the "eu-west-1" region.
$ eksctl create cluster -n zerotwo -r eu-west-1 --profile [AWS-PROFILE-NAME]
