Initial Setup

Install Foundation‚Äã

  1. Install Docker.
  2. Download fn from https://github.com/namespacelabs/foundation/releases/. Make sure to pick the right binary for your CPU architecture and operating system, unpack the archive, and place the binary somewhere you can access.

Clone the Example Project‚Äã

git clone https://github.com/namespacelabs/examples
cd examples/todos

Accessing our Private Repositories‚Äã

info

Foundation repositories are private during our early access, so you need to be authenticated to Github when downloading dependencies.

Foundation assumes by default that you use SSH to authenticate with Github. And if you do, we'll set things up so that the private repositories can be accessed by your workstation. But if that's not the case, pass --git_ssh_auth=false when running fn.

Initialize the Project‚Äã

Foundation manages development dependencies on your behalf, including required SDK versions, supporting infrastructure, etc.

To get started, prepare your development environment by running:

fn prepare

This sets up a series of containers in your Docker installation, that are used to support the development workflow. These include Buildkit (used for various build operations), and k3s (for a local Kubernetes development cluster) and a Docker registry used for local image management. Most of the time you're not interacting with these, we do it for you. Including updating their versions over time as necessary.

We then update a local devhost.textpb with some information on what we setup. This configuration file supports per-environment stanzas, but we'll get to that later.

You need to run fn prepare for every project root, but the subsequent executions are fast as we don't re-do work previously done.

Time to do some development on the app!
